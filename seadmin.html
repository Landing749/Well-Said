Firebase DB Rules

Copy and paste these rules into your Firebase Realtime Database console.

Realtime Database Rules
{ "rules": { "inquiries": { ".read": "auth != null", ".write": "auth != null" }, "chats": { "$code": { ".read": true, ".write": true, "$msgId": { ".validate": "newData.hasChildren(['sender', 'text', 'ts']) && newData.child('sender').isString() && newData.child('text').isString() && newData.child('text').val().length <= 2000" } } }, "codes": { ".read": true, ".write": "auth != null" }, "maintenance": { ".read": true, ".write": "auth != null" } } }
Rule Explanations

/inquiries — Auth only

Only Firebase authenticated admins can read or write inquiry data.

/chats/{code} — Public (code is the secret)

Chat rooms are keyed by session code. Knowing the code path IS the access. Messages are validated for required fields and max 2000 chars.

/codes — Public read, auth write

Clients can look up their code. Only admins can create code entries.

/maintenance — Public read, auth write

Anyone can check maintenance status. Only admins can change it.
